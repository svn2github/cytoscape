CytoscapeEditor Release Notes (version 2.60)

  1. Patched Editor so that mouse, key, and drop events are ignored when the Editor tab of the CytoPanel is not active.

  2. Fixed the "ESC" key to correctly stop drop of an edge.

Prior Notes

CytoscapeEditor Release Notes (version 2.53)

  1. Fixed bug where palette was being redrawn when the Editor tab wasn't selected.

  2. Some code cleanup changes to CytoscapeEditorManagerSupport.

CytoscapeEditor Release Notes (version 2.52)

  1. Changed File->New-Network->EmptyNetwork (NewNetworkAction) to not produce an extra (accidental) CyNetworkView and to set the visual style of the new network to be the VisualMappingManager's current visual style.

  2. Removed arrowhead on default "Directed Edge" edge palette entry in DefaultCytoscapeEditor.

  3. Fixed so that clicking on the Editor tab would correctly show the correct editor palette for a given visual style (for sublass editors).

    This required stopping the creation of a CyNetworkView in CytoscapeEditorManagerSupport.onComponentSelected() that would be set to the default visual style thereby resetting the editor palette for subclasses (like the HyperEdgeEditor).

  4. Changed call to buildVisualStyle() in CytoscapeEditorManager.initializeEditor() to occur before VisualMappingManager.setVisualStyle() is called.

CytoscapeEditor Release Notes (version 2.51)

  1. Simplified CytoscapeEditor initialization with assumption that the CytoscapeEditor is always loaded before any extension to this editor.

  2. Removed unneeded definitions of getPluginInfoObject() and describe().

  3. Removed manual setting of EdgeView tooltips, since tooltips should now be set using Visual Styles.

  4. Fixed NPE bug in CyotscapeEditorManger.register() where CytoscapeEditorManager.log() statements were being called without null pointer check.

  5. Updated missed deprecated code reference to use new Cytoscape 2.5 VizMap API.

CytoscapeEditor Release Notes (version 2.50)

  1. Made CytoscapeEditor be a good citizen for the Cytoscape PluginManager.

    Added CytoscapeEditorPlugin.getPluginInfoObject().

  2. Updated code to use new Cytoscape 2.5 VizMap API.

  3. Fixed various deprecations and warnings.

CytoscapeEditor Release Notes (version 2.4 alfa 4)

    2. Cleanup and changed BasicCytoscapeEditor.addNodeContextMenuItems(), addEdgeContextMenuItems(), and removeExistingDeleteMenuItemIfNecessary() to handle changes to DeleteAction.

CytoscapeEditor Release Notes (version 2.4 alfa 2)

  1. Changed the CytoscapeEditor to be intelligent about what drag cursor is shown when performing a drag-drop from a shape palette entry to a CyNetworkView of interest.

    Built a flexible framework using DragSourceContextCursorSetters to determine cursor for each palette entry.

CytoscapeEditor Release Notes (version 2.4 alfa 1)

  2. Cleaned up code and refactored PaletteNetworkEditEventHandler.itemDropped() and associated methods for easier use by subclasses and code cleanup.

CytoscapeEditor Release Notes (version 2.3.03)

  1. Created a new way to easily produce ShapePalette entries for editors.

    This involves the use of new interfaces ShapePaletteInfo and ShapePaletteInfoGenerator and their supporting classes. Refactored DefaultCytoscapeEditor to use this new ShapePaletteInfoGenerator.

  2. Added ability of ShapePalette entries to handle visual style size information.

    Changed computation of BasicCytoShapeEntity size to allow for larger CytoShapeIcons. Added CytoShapeIcon constructor for handling size information. Changed getIconWidth/Height() to return real width and height of icon. Changed the ShapePalette JList to no longer use setFixedCellHeight() since BasicCytoShapeEntitys can now have different sizes.

  3. Added CytoShapeIcon.DEFAULT_WIDTH and DEFAULT_HEIGHT and deprecated CytoShapeIcon.WIDTH & HEIGHT.

  4. Removed SimpleBioPAX_Editor.
