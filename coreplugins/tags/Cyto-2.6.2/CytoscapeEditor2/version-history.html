CytoscapeEditor Version History

  • CytoscapeEditor, Version 2.62 (01/25/09)
    1. Ctrl-click on a one-button Mac brings up both the bypass menu AND creates an edge (1738)
    2. Connect Selected Nodes not undoable as an atomic action. (1384)
    3. "Connect Selected Node" not always show up (1264)
    4. Creating new edges via CTRL-click opens interfering menu on Mac (1806)
    5. Creating new edge leaves streaks on canvas (still) (1730)
    6. CytoscapeEditor in 2.6 throws exception when adding (globally) existing node (1917)
    7. Fixed ClassCastException where using the VizMapper and changing ArrowShapes could lead to a ClassCastException.
  • CytoscapeEditor, Version 2.60 (07/09/08)
    1. Patched Editor so that mouse, key, and drop events are ignored when the Editor tab of the CytoPanel is not active.
    2. Fixed the "ESC" key to correctly stop drop of an edge.
  • CytoscapeEditor, Version 2.53 (10/30/07)
    1. Fixed bug where palette was being redrawn when the Editor tab wasn't selected.
    2. Some code cleanup changes to CytoscapeEditorManagerSupport.
  • CytoscapeEditor, Version 2.52 (10/25/07)
    1. Changed File->New-Network->EmptyNetwork (NewNetworkAction) to not produce an extra (accidental) CyNetworkView and to set the visual style of the new network to be the VisualMappingManager's current visual style.
    2. Removed arrowhead on default "Directed Edge" edge palette entry in DefaultCytoscapeEditor.
    3. Fixed so that clicking on the Editor tab would correctly show the correct editor palette for a given visual style (for sublass editors).
    4. Changed call to buildVisualStyle() in CytoscapeEditorManager.initializeEditor() to occur before VisualMappingManager.setVisualStyle() is called.
  • CytoscapeEditor, Version 2.51 (07/23/07)
    1. Simplified CytoscapeEditor initialization with assumption that the CytoscapeEditor is always loaded before any extension to this editor.
    2. Removed unneeded definitions of getPluginInfoObject() and describe().
    3. Removed manual setting of EdgeView tooltips, since tooltips should now be set using Visual Styles.
    4. Fixed NPE bug in CyotscapeEditorManger.register() where CytoscapeEditorManager.log() statements were being called without null pointer check.
    5. Updated missed deprecated code reference to use new Cytoscape 2.5 VizMap API.
  • CytoscapeEditor, Version 2.501 (06/30/07)
    1. Updated missed deprecated code reference to use new Cytoscape 2.5 VizMap API.
  • CytoscapeEditor, Version 2.50 (05/11/07)
    1. Made CytoscapeEditor be a good citizen for the Cytoscape PluginManager.
    2. Updated code to use new Cytoscape 2.5 VizMap API.
    3. Fixed various deprecations and warnings.
  • CytoscapeEditor, Version 2.4 alfa 4 (01/09/07)
    1. Added keyboard accelerator for "Delete Selected Nodes and Edges" menu item.
  • CytoscapeEditor, Version 2.4 alfa 3 (12/29/06)
    1. Various changes and cleanup to allow easier subclassing of CytoscapeEditor and access to the DeleteAction.
  • CytoscapeEditor, Version 2.4 alfa 2 (12/17/06)
    1. Changed the CytoscapeEditor to be intelligent about what drag cursor is shown when performing a drag-drop from a shape palette entry to a CyNetworkView of interest.
  • CytoscapeEditor, Version 2.4 alfa 1 (12/08/06)
    1. Refactored BasicNetworkEditEventHandler.finishEdge() into smaller pieces for subclass usage. Cleaned up BasicNetworkEditEventHandler.mousePressed() and a few protected instance variables that should be private.
    2. Cleaned up code and refactored PaletteNetworkEditEventHandler.itemDropped() and associated methods for easier use by subclasses and code cleanup.
    3. Removed BioPAXNetworkEditEventHandler.
  • CytoscapeEditor, Version 2.3.03 (12/05/06)
    1. Created a new way to easily produce ShapePalette entries for editors.
    2. Added ability of ShapePalette entries to handle visual style size information.
    3. Added CytoShapeIcon.DEFAULT_WIDTH and DEFAULT_HEIGHT and deprecated CytoShapeIcon.WIDTH & HEIGHT.
    4. Removed SimpleBioPAX_Editor.
  • CytoscapeEditor, Version 2.3.02 (11/06/06)
    1. Fixed compilation problem caused by accidental release of changes to CytoscapeEditorManager.java.
  • CytoscapeEditor, Version 2.3.01 (09/13/06)
    1. CytoscapeEditorManagerSupport: Fixed updateEditorPalette() to not assume visualStyleName is the same as Editor name. Added some comments.
    2. BasicCytoscapeEditor: Removed setting of BIO_PAX attributes from addNode(). Fixed deprecated method usage in addNode().
    3. BasicCytoShapeEntity, ShapePalette, InvalidEditorException: Added generated serial version UUID for serializable classes. CytoscapeEditor: Updated to remove deprecated use of FlagEventListener (now is SelectEventListener)
    4. CytoscapeEditorManager:
      1. Fixed so visualStyleName can be different from the editor name.
      2. Added getEditorNameForVisualStyleName() and changed initializeEditor() to take a visualStyleName parameter.
      3. Changed many instance variables to be private vs protected.
      4. Added intialization of this manager if register() is called by another plugin before this plugin is loaded.
    5. CytoscapeEditorPlugin:
      1. Removed deprecated call to CytoscapeInit.getDefaultVisualStyle() in initializeCytoscapeEditor().
      2. Removed some misleading code--println that editor is loaded and initialization code as though editor is initialized via a menu item, when it is not.

This page was last updated: 01/25/09
